# 本地预设工具栏重构计划

## 背景

当前“本地预设”头部已经具备搜索、多选、检查更新等能力，但视觉上仍然存在明显问题：

1. 标题、说明、批量按钮、搜索入口混在同一层级，信息主次不清。
2. 搜索展开会改变工具栏布局，容易造成标题区和列表整体观感不稳定。
3. 多选模式进入后，头部会出现“解释文字 + 批量动作”同时争抢空间的问题，造成中间空、左右散。
4. 低频功能（排序）不适合以大面积原生下拉框直接占据主工具栏。
5. 批量模式的文案过长，用户更需要直接动作，而不是额外说明。

## 参考原则

这次重构不照搬单个产品，而是吸收几类成熟桌面软件的共性做法：

1. Apple `UIToolbar`
   链接: https://developer.apple.com/documentation/uikit/uitoolbar
   原则: 工具栏承载的是当前视图最相关的动作；重要动作要突出，普通动作不要和主动作抢视觉权重。

2. Figma UI3
   链接: https://help.figma.com/hc/en-us/articles/23954856027159-Navigating-UI3
   原则: 动作应按上下文自适应；选择态出现后，展示“当前选择最相关的动作”，而不是堆更多说明文字。

3. VS Code View Toolbar
   链接: https://code.visualstudio.com/docs/configure/custom-layout
   原则: 工具栏不能太忙；低频动作应该进入 `...` 更多菜单，而不是长期占据主栏。

## 目标布局

重构后的“本地预设”头部采用两层、三段式结构：

1. 第一层：标题层
   左侧只放区块标题与图标。
   右侧只放常驻动作。

2. 第二层：状态层
   默认状态下只放一行极短说明 + 悬浮说明按钮。
   多选状态下不再显示“多选模式”“已选中多少个预设”之类说明，只显示三个动作：`全选`、`反选`、`删除`。

3. 右侧动作分层
   常驻动作保留：`搜索`、`多选`、`更多`、`检查预设`。
   低频动作“排序方式”移入 `更多` 菜单，不再直接显示原生下拉框。

## 交互方案

### 默认状态

1. 左侧显示：
   `本地预设`
   `已下载到本机的配置`
   `?` 悬浮说明

2. 右侧显示：
   搜索按钮
   多选按钮
   更多按钮
   检查预设按钮

3. 搜索框改成浮层，不再把左侧文字往下挤。

### 多选状态

1. 标题层保持稳定，不新增整条工具栏。
2. 第二层左侧说明隐藏。
3. 第二层右侧只显示：
   `全选`
   `反选`
   `删除`
4. `删除` 保持危险动作颜色，但文字只保留“删除”两个字。
5. 右侧常驻动作收敛：
   保留多选切换按钮用于退出。
   隐藏“检查预设”这类与当前选择无关的动作。
   搜索不再占用布局空间。

## 具体实施清单

1. 重构 `index.html`
   重写本地预设头部 DOM，拆成标题层和状态层。
   加入 `更多` 按钮和排序菜单容器。
   把搜索输入从 inline 展开改成浮层展开。

2. 重构 `presets.js`
   更新 `updateLocalManagerUI()` 的状态机。
   让默认态和多选态分别驱动不同动作区。
   新增本地工具栏更多菜单的打开、关闭、排序切换逻辑。
   多选状态文案缩短为 `全选 / 反选 / 删除`。

3. 重构 `style.css`
   用 grid/flex 重新组织头部结构。
   搜索框改为悬浮定位，不参与主布局宽度计算。
   按钮高度、对齐、留白统一。
   批量动作做轻量淡入，不影响列表垂直位置。

## TDD / 验证项

### 结构 smoke

1. 旧的下方批量工具行不存在。
2. 排序原生下拉不再直出在主工具栏。
3. 头部存在 `更多` 按钮和排序菜单结构。
4. 搜索输入位于浮层容器中，而不是主栏 inline 撑开。

### 交互 smoke

1. 多选模式下不出现“多选模式”等长文案。
2. 多选按钮区只保留 `全选 / 反选 / 删除`。
3. 多选模式下“检查预设”隐藏。
4. 删除按钮在未选中时禁用，但文案仍为“删除”。

### 手工联调

1. 默认状态下头部视觉重心要回到左标题、右动作，不再出现中间大片悬空。
2. 打开搜索时，左侧标题和说明不移动。
3. 进入多选时，卡片列表首项不会整体下跳。
4. 头部在 1280px 左右窗口下依然完整，不出现莫名断行。
