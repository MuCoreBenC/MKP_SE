## 1. 文档目标

本文件用于把路线图和里程碑继续拆成可执行任务积压清单。  
目标不是替代 issue 系统，而是在当前阶段给出统一优先级、依赖关系和推进顺序。

---

## 2. 使用规则

1. 所有新任务必须归属到某个领域
2. 所有高风险任务必须绑定测试和验收标准
3. 没有明确依赖和退出条件的任务不能进入当前迭代
4. 不允许把“临时修一下”当作长期任务描述
5. 影响用户安装包体积、状态真相源、更新链路的任务默认高优先级

---

## 3. 优先级定义

### P0

必须优先处理，否则项目继续开发会持续放大风险。

### P1

重要但可以建立在 P0 稳定之后处理。

### P2

增强项或局部体验优化，不应阻塞核心闭环。

---

## 4. 领域划分

实施任务按以下领域管理:

- B1 状态与同步
- B2 预设与资源
- B3 参数与校准
- B4 更新与发布
- B5 用户端体验
- B6 开发者工具
- B7 测试与质量
- B8 包体与构建
- B9 日志与诊断
- B10 文档与流程

---

## 5. 当前总优先级结论

当前阶段总体顺序固定为:

1. B1 状态与同步
2. B2 预设与资源
3. B3 参数与校准
4. B4 更新与发布
5. B8 包体与构建
6. B7 测试与质量
7. B5 用户端体验
8. B6 开发者工具
9. B9 日志与诊断
10. B10 文档与流程

说明:

- B6 虽然重要，但前提是 B4 边界先稳定
- B5 不能再凌驾于 B1 到 B4 之上

## 6. Backlog 明细

### B1 状态与同步

#### B1-001 盘点全部持久化 key

- 优先级: P0
- 目标: 建立当前 `localStorage` / 文件配置 / 运行态 key 清单
- 依赖: 无
- 验收:
  - 所有 key 有来源说明
  - 所有 key 有用途说明
  - 标记可迁移 / 待废弃 / 必保留

#### B1-002 建立 `UserConfigStore`

- 优先级: P0
- 目标: 用户配置统一经 Store 读写
- 依赖: B1-001
- 验收:
  - 页面层不再直接写关键用户配置
  - 多窗口能收到配置变更事件

#### B1-003 建立 `PresetSessionStore`

- 优先级: P0
- 目标: 统一当前品牌 / 机型 / 版本 / 预设上下文
- 依赖: B1-001
- 验收:
  - 不再出现 `_null` 作为正常运行上下文
  - 各页面当前预设上下文一致

#### B1-004 建立跨窗口同步协议

- 优先级: P0
- 目标: 统一广播、订阅、冲突提示规则
- 依赖: B1-002, B1-003
- 验收:
  - 另一窗口修改当前预设时能提示刷新
  - 草稿状态不会被静默覆盖

### B2 预设与资源

#### B2-001 统一预设元信息解析

- 优先级: P0
- 目标: 从 JSON 内容提取版本和类型，而不是依赖文件名
- 依赖: B1-003
- 验收:
  - 重命名文件不影响最新版本判断
  - 同一内容不同文件名结果一致

#### B2-002 建立 `PresetService`

- 优先级: P0
- 目标: 统一本地、默认、远端预设读取与应用
- 依赖: B2-001
- 验收:
  - 下载页、本地页、当前应用状态一致
  - 应用预设后全局上下文刷新一致

#### B2-003 明确恢复默认来源

- 优先级: P0
- 目标: 定义恢复默认回到哪一层资源
- 依赖: B2-002
- 验收:
  - 恢复默认行为有文档说明
  - 用户离线时仍能恢复到安装包默认资源

#### B2-004 本地预设管理修正

- 优先级: P1
- 目标: 修正本地显示、映射、排序、当前应用标记
- 依赖: B2-002
- 验收:
  - 本地文件存在时 UI 和实际文件一致
  - 不再出现重复“当前应用”

### B3 参数与校准

#### B3-001 建立 `ParamEditorService`

- 优先级: P0
- 目标: 文本模式和分行模式共享同一真相源
- 依赖: B1-003
- 验收:
  - 两种编辑模式切换不丢状态
  - 保存逻辑统一

#### B3-002 建立全局 undo/redo 策略

- 优先级: P0
- 目标: 参数页文字修改、插行、删行共享同一撤销历史
- 依赖: B3-001
- 验收:
  - 文本修改可撤销
  - 插行 / 删行可撤销
  - 模式切换后撤销历史仍可用

#### B3-003 建立 `CalibrationService`

- 优先级: P0
- 目标: 统一 Z / XY 当前值与修改值来源
- 依赖: B3-001
- 验收:
  - 参数变更后校准页同步刷新
  - 恢复默认后校准页同步刷新

#### B3-004 草稿保护和切页拦截

- 优先级: P0
- 目标: 未保存修改切页时统一提示
- 依赖: B3-001
- 验收:
  - 保存并切换 / 不保存 / 取消 三条路径清晰
  - 不保存时不写入持久层

### B4 更新与发布

#### B4-001 固定本地版本源

- 优先级: P0
- 目标: 统一当前运行版本来源，避免多处版本定义冲突
- 依赖: 无
- 验收:
  - 手动检查与页面显示版本一致
  - 红点逻辑与版本页逻辑一致

#### B4-002 固定远端 manifest 读取规则

- 优先级: P0
- 目标: 统一远端 manifest 获取、缓存、失败回退
- 依赖: B4-001
- 验收:
  - 远端失败时有明确降级策略
  - 不再因为 manifest 异常整页空白

#### B4-003 稳定 patch 应用与回退

- 优先级: P0
- 目标: 明确下载、应用、回退、日志链路
- 依赖: B4-001, B4-002
- 验收:
  - 应用失败能记录日志
  - 回退后仍能正常启动旧版本

#### B4-004 剥离发布后台

- 优先级: P0
- 目标: 用户端默认构建中不包含发布后台页面和依赖
- 依赖: B4-001, B8-001
- 验收:
  - 用户安装包中无后台页面入口
  - 后台工具可独立构建

### B5 用户端体验

#### B5-001 首页展示模式重构

- 优先级: P2
- 目标: 在不破坏数据边界的前提下优化品牌 / 机型展示
- 依赖: B1-003
- 验收:
  - 简洁模式和详细模式不依赖重复数据
  - 右键功能不破坏默认目录数据

#### B5-002 预设文件管理体验增强

- 优先级: P2
- 目标: 完善搜索、排序、多选、右键体验
- 依赖: B2-002
- 验收:
  - 操作和真实文件状态一致
  - 多选状态稳定

#### B5-003 参数页视觉增强

- 优先级: P2
- 目标: 优化列表感、分组感、提示样式
- 依赖: B3-001, B3-003
- 验收:
  - 不影响保存、撤销、切页拦截

### B6 开发者工具

#### B6-001 独立发布工具工程入口

- 优先级: P1
- 目标: 为发布后台建立独立构建入口
- 依赖: B4-004
- 验收:
  - 可独立运行
  - 与用户端共享规则，不共享页面实现

#### B6-002 默认资源管理能力沉淀

- 优先级: P1
- 目标: 品牌、机型、图片、预设维护进入独立工具
- 依赖: B6-001
- 验收:

### B7 测试与质量

#### B7-001 建立最小单元测试集

- 优先级: P0
- 目标: 覆盖版本比较、manifest 解析、预设元信息提取
- 依赖: 无
- 验收:
  - 能独立执行
  - 至少覆盖高风险纯逻辑

#### B7-002 建立核心冒烟清单

- 优先级: P0

- 依赖: 无
- 验收:
  - 首次启动、应用预设、参数保存、检查更新均在清单内

#### B7-003 建立多窗口专项验证

- 优先级: P1
- 目标: 把同步和冲突提示纳入回归
- 依赖: B1-004
- 验收:

### B8 包体与构建

#### B8-001 建立用户端构建白名单

- 优先级: P0

- 依赖: 无
- 验收:
  - `references`、`scripts`、历史 patch、后台工具不进入用户包

#### B8-002 建立安装包体积门禁

- 优先级: P0

- 依赖: B8-001

### B9 日志与诊断

- 依赖: B4-003

  - 日志字段一致

#### B9-002 建立诊断导出能力

- 目标: 用户可导出最小诊断包
- 依赖: B9-001
- 验收:
  - 不包含隐私画像数据
  - 能辅助排查更新和配置异常

- 目标: 把版本、体积、冒烟、上传步骤固定化
- 依赖: B4-003, B8-002
- 验收:
  - 发版前后步骤有统一模板

---

## 9. 文档结论

这份 Backlog 的价值不在于把任务列得很多，而在于强制回答三个问题:

- 这个任务属于哪个领域
- 它的前置依赖是什么
- 它会不会再次破坏当前项目最脆弱的边界

只要这三个问题持续被约束住，后面的开发就不会再退化成单点救火。
